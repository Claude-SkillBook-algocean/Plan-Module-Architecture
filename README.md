# 📐 Plan-Module-Architecture

**계층적 단일책임 모듈 설계 (Hierarchical SRP Module Architecture Design)**

프로젝트 개발 전, 철저한 모듈 설계를 수행하는 Claude 스킬입니다.  
기술스택 선정부터 계층적 모듈 분해, 상세 IN/OUT 명세, 메인 파이프라인 설계까지 진행하고,  
최종 결과물을 **인터랙티브 HTML 설계서**로 생성합니다.

---

## 🎯 핵심 철학

> **"하나의 모듈은 하나의 책임만, OUT은 반드시 1개, 메인 파이프라인은 직선 1줄"**

```
┌─────────────────────────────────────────────────┐
│  모듈 ABC → 기능 ABC 담당                         │
│                                                 │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐   │
│  │ 모듈 A    │    │ 모듈 B    │    │ 결합 모듈  │   │
│  │ In: a1   │───▶│ In: A    │───▶│ In: B    │──▶│ Out: ABC
│  │ Out: A   │    │ In: api  │    │          │   │
│  └──────────┘    │ Out: B   │    └──────────┘   │
│       ▲          └────┬─────┘         ▲         │
│       │               │              │         │
│  In: tdwz        ┌────┴─────┐        │         │
│                  │ 공통 C1   │────────┘         │
│                  │ In: .env  │                  │
│                  │ Out: api  │                  │
│                  └──────────┘                  │
└─────────────────────────────────────────────────┘
```

## 🔄 워크플로우

이 스킬은 반드시 다음 순서로 진행됩니다:

### Phase 1: 기술스택 협의
- 프로젝트 목적, 도메인, 환경을 파악
- 기술스택 후보를 표로 제안
- **사용자 승인 후** 다음 단계 진행

### Phase 2: 모듈 계층 설계
- 프로젝트를 Feature 단위(F1, F2...)로 분해
- 각 Feature를 최소 단위 모듈(F1.1, F1.2...)로 분해
- 공통 모듈(C0) 설계
- SRP 검증: "변경 이유가 2가지 이상이면 분할"

### Phase 3: 상세 IN/OUT 명세
- 모든 모듈에 파라미터명, 타입, 출처, 설명 명시
- OUT은 필드 레벨까지 구조 기재
- 메인 파이프라인의 단계별 IN→OUT 체인 연결

### Phase 4: HTML 설계서 생성
- 다크 테마 인터랙티브 HTML
- 접기/펼치기, 목차 네비게이션, 색상 코딩
- 설계서만 보고 바로 구현 가능한 수준의 상세도

---

## 📏 모듈 설계 핵심 규칙

| 규칙 | 설명 |
|:-----|:-----|
| **OUT = 1** | 하나의 모듈은 반드시 1개의 OUT만 가진다 |
| **IN ≥ 1** | IN은 1개 이상, 메인 IN과 보조 IN으로 구분 |
| **파이프라인 = 직선** | 메인 파이프라인은 분기 없는 1줄 |
| **SRP 엄수** | 변경 이유가 2가지 이상이면 반드시 분할 |
| **최하위 = 원자적** | 최하위 모듈은 단일 기능만 수행 |
| **명세 = 코드 수준** | IN/OUT에 함수명, 변수명, 타입 모두 기재 |
| **공통모듈 독립** | C0은 Feature와 분리, DI(의존성 주입)로 전달 |

---

## 🎨 HTML 설계서 특징

- **다크 테마** — 눈의 피로 최소화
- **색상 코딩** — IN(녹색), OUT(주황), API(시안), Feature별 고유색
- **폰트** — JetBrains Mono(코드) + Noto Sans KR(본문)
- **인터랙티브** — 모듈 접기/펼치기, 전체 토글, 목차 네비게이션
- **파이프라인 시각화** — 직선형 흐름도 + 단계별 IN/OUT 테이블
- **의존성 매트릭스** — 모듈 간 참조 관계 한눈에 파악
- **구현 체크리스트** — 체크박스로 진행 추적

## 📋 대규모 프로젝트 지원

모듈 30개 이상의 대규모 프로젝트인 경우:
1. C0 → F1 → F2 → ... 순서로 Feature 단위로 나누어 작성
2. 각 Feature 완료 시 사용자 확인 후 다음 진행
3. 매 응답에 `[현재: FN 완료 / 다음: FM 예정]` 진행 상황 표시
4. 최종적으로 하나의 HTML로 통합
